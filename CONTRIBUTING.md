# Contributing to SongDrop

First off, thank you for considering contributing to SongDrop! 🎵

## How Can I Contribute?

### 🐛 Reporting Bugs

If you find a bug, please create an issue with:
- A clear, descriptive title
- Steps to reproduce the behavior
- Expected vs actual behavior
- Screenshots if applicable
- Browser and OS information

### 💡 Suggesting Features

Feature suggestions are welcome! Please:
- Check if the feature has already been suggested
- Provide a clear description of the feature
- Explain why it would be useful
- Include mockups or examples if possible

### 🎨 Design Improvements

Have ideas for better UI/UX?
- Create an issue with "Design:" prefix
- Include mockups, color schemes, or references
- Explain the reasoning behind the change

### 📝 Code Contributions

#### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/songdrop.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit: `git commit -m "Add: your feature description"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

#### Code Style Guidelines

- Use clear, descriptive variable names
- Add comments for complex logic
- Keep functions focused and small
- Follow existing code formatting
- Test in multiple browsers (Chrome, Firefox, Safari)

#### Commit Message Format

```
Type: Brief description

- Detailed point 1
- Detailed point 2
```

**Types:**
- `Add:` New feature or functionality
- `Fix:` Bug fix
- `Update:` Improve existing feature
- `Refactor:` Code restructuring
- `Docs:` Documentation changes
- `Style:` Formatting, CSS changes

**Examples:**
```
Add: Support for SoundCloud links

- Parse SoundCloud track URLs
- Fetch metadata via API
- Update platform badge styling
```

```
Fix: iPod color picker not updating wheel

- Bind click handler to color swatches
- Update CSS variable on selection
- Test with all 10 color options
```

### 📚 Documentation

Documentation improvements are always welcome:
- Fix typos or unclear instructions
- Add examples or screenshots
- Improve setup guides
- Translate to other languages

### 🧪 Testing

Help test new features:
- Test on different devices (mobile, tablet, desktop)
- Test on different browsers
- Test edge cases (very long titles, special characters, etc.)
- Report any issues found

## Pull Request Process

1. **Update documentation** if you changed functionality
2. **Test thoroughly** in multiple browsers
3. **Keep PR focused** — one feature/fix per PR
4. **Write clear description** of what and why
5. **Link related issues** using `Fixes #123` or `Closes #456`
6. **Be patient** — reviews may take a few days

## Questions?

Feel free to:
- Open an issue with the "question" label
- Reach out via discussions
- Check existing issues and PRs for answers

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Assume good intentions
