# 🚀 Deployment Checklist

Complete this checklist to ensure a smooth deployment of SongDrop.

---

## 📦 Pre-Deployment

- [ ] All files downloaded from Claude
- [ ] Files organized in a folder on your computer
- [ ] GitHub account created and verified
- [ ] Decided on repository name (`songdrop` recommended)

---

## 🏗️ Repository Setup

- [ ] Created new repository on GitHub
- [ ] Set visibility to **Public**
- [ ] **DID NOT** initialize with README (using provided one)
- [ ] Selected **MIT License**
- [ ] Uploaded all files:
  - [ ] `index.html`
  - [ ] `README.md`
  - [ ] `SETUP.md`
  - [ ] `LICENSE`
  - [ ] `CONTRIBUTING.md`
  - [ ] `QUICK-REFERENCE.md`
  - [ ] `.gitignore`

---

## 🌐 GitHub Pages Setup

- [ ] Navigated to Settings → Pages
- [ ] Set Source to **Branch: main**
- [ ] Set Folder to **/ (root)**
- [ ] Clicked Save
- [ ] Waited 1-2 minutes for deployment
- [ ] Green banner appeared with live URL

---

## ✏️ Customization

- [ ] Updated `README.md` — replaced all `yourusername` with actual username
- [ ] Updated `index.html` meta tags — replaced URLs with actual GitHub Pages URL
- [ ] Updated About section in repository with:
  - [ ] Description
  - [ ] Website URL
  - [ ] Topics/tags
- [ ] Tested live site in browser
- [ ] Tested on mobile device

---

## 🧪 Testing

- [ ] Created a test SongDrop
- [ ] Verified iPod displays correctly
- [ ] Tested background themes
- [ ] Tested iPod colors
- [ ] Tested sticky note feature
- [ ] Tested on different browsers:
  - [ ] Chrome/Edge
  - [ ] Firefox
  - [ ] Safari
  - [ ] Mobile browser
- [ ] Shared link works when opened in incognito/private mode
